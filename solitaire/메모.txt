이슈
    Card.gd, move_manager.gd 같은 카드를 두 번 눌러서 선택을 해제하는 기능을 넣으려 하는데 현재 같은 카드를 두 번 누르면 두 번째는 인식되지 않음.
    클릭 수행 중 가끔 정상적이지 않은 목표가 선택됨(가려져 있는 뒤쪽이 선택됨)


변경점:
    card의 불필요한 is_top_card 함수 제거
    카드 스프라이트 표시에서 같은 레벨에서 if문 여럿을 사용하면서 자동으로 else문이 읽히던 것을 elif로 수정하여 고침
    뒷면 표시된 카드는 어차피 다룰 수 없기 때문에 CollisionShape2d를 비활성화하도록 하여 겹쳐진 목표가 선택되는 빈도를 줄임
    0카드, 즉 스택의 위치 표시용 카드 위에 카드가 존재할 경우 collisionshape2d를 비활성화하도록 하여 잘못된 목표가 선택되는 빈도를 줄임

메모
-2 : 파운데이션
-1 : 덱         -1은 선택 시 덱에서 카드를 하나 배치
0 : 빈공간       0은 이동의 목표로만 선택 가능하며 K만이 이동 가능

스톡
1. 스톡의 위치에 스위치 역할을 하는 뒷면 표시의 카드를 하나 배치(선택은 불가능한 카드)
2. 해당 카드를 누르면 스톡의 스택에서 하나 팝해서 측면에 카드를 앞면으로 배치.
3. 앞면으로 배치된 카드는 테이블 카드와 같은 이동규칙을 지님.
(또한 prev_card를 지니며, 이 prev_card는 이동에 대한 제한과 무관함)
4. 스톡이 빈 경우 스톡 뒷면 표시 카드는 없애거나, 색을 바꾸는 등 표시
5. 스톡이 빈 경우 해당 카드를 누르면 측면에 꺼내 놓았던 카드를 위에서부터 순서대로 다시 스톡 스택에 저장(prev_card 이용, 역순으로 집어넣으면서 제거)

진행중
    덱 카드 액세스
        클릭 시 1장, 덱 옆으로 이동 후 앞면으로 표시.
        추가로 클릭 시, 같은 위치에 이동 후 앞면 표시.
        남은 카드가 0장이 되면 모든 남은 카드를 돌려보내고 뒷면 표시로 전환한다.
        이동규칙은 테이블과 같다.
